# React Component Standards

## Import Order
Always import in this sequence:
1. React and hooks from 'react'
2. Third-party libraries (redux, react-router, etc.)
3. Local components from relative paths
4. CSS Modules

```jsx
import React, { useState } from 'react';
import { useAppDispatch, useAppSelector } from '../store/hooks';
import BookCard from './BookCard';
import styles from './BookList.module.css';
```

## Component Definition
- **Only functional components with hooks** (no class components)
- Destructure props in function signature with default values where appropriate

```jsx
function BookList({ books = [], isLoading = false }) {
  // component logic
}
```

## State Management

### Redux Toolkit Pattern (✓ Correct)
Use `useAppDispatch` and `useAppSelector` from `../store/hooks` for shared state:
```jsx
const dispatch = useAppDispatch();
const favorites = useAppSelector(state => state.favorites.items);

// Dispatch Redux Toolkit slices
dispatch(addFavorite(bookId));
```

### Anti-Pattern (✗ Incorrect)
Never use raw hooks from 'react-redux':
```jsx
// ✗ DON'T DO THIS
import { useDispatch, useSelector } from 'react-redux';
const dispatch = useDispatch();
```

Use `useState` only for local UI state (form inputs, dropdowns, modals).

## Styling
- Import CSS Modules: `import styles from '../styles/ComponentName.module.css'`
- Reference classes as properties: `className={styles.container}`

## Authentication
Always verify token before dispatching authenticated API calls. Redirect to login if missing:
```jsx
if (!token) {
  navigate('/login');
  return;
}
dispatch(fetchUserData());
```

## Rules Summary
- ✓ Functional components + hooks only
- ✓ Correct import order
- ✓ Destructure props with defaults
- ✓ useAppDispatch/useAppSelector from ../store/hooks
- ✓ Redux Toolkit slices for shared state
- ✓ useState for local UI state only
- ✓ CSS Modules with styles.className pattern
- ✓ Check token before authenticated API calls
